<p>With the <a href="https://blog.qgis.org/2021/05/04/qgis-grant-programme-2021-results/">QGIS Grant Programme 2021</a>, we were able to support eight proposals that are aimed to improve the QGIS project, including software, infrastructure, and documentation. The following reports summarize the work performed in the proposals. </p>
<ol>
<li class="task-list-item enabled position-relative border-right-0"><strong> </strong><span class="reference"><a class="issue-link js-issue-link" href="https://github.com/qgis/QGIS-Enhancement-Proposals/issues/213"><strong>QGIS Server and services documentation</strong><span class="issue-shorthand"><strong> (#213)</strong></span></a></span> &#8211; <a href="https://lists.osgeo.org/pipermail/qgis-psc/2022-January/009626.html" rel="nofollow">Report</a><br />The Services chapter of the QGIS Server documentation needed some love to<br />be effectively representative of the underlying implementation. Numerous<br />services, requests or parameters were not documented at all. Some others<br />also had very sketchy descriptions. Thanks to this QEP, the Services<br />chapter is now in a much better shape!</li>
<li class="task-list-item enabled position-relative border-right-0"><span class="reference"><a class="issue-link js-issue-link" href="https://github.com/qgis/QGIS-Enhancement-Proposals/issues/216"><strong>Rework handling of multi-layer, mixed-format datasets<span class="issue-shorthand"> (#216)</span></strong></a></span> &#8211; <a href="https://lists.osgeo.org/pipermail/qgis-psc/2021-July/009429.html" rel="nofollow">Report</a><br />While the work was partly motivated as an opportunity to clean up some<br />older parts of the QGIS codebase which were fragile and had low test<br />coverage, it has also resulted in many improvements and polish in the<br />QGIS user interface.</li>
<li class="task-list-item enabled position-relative border-right-0"><span class="reference"><a class="issue-link js-issue-link" href="https://github.com/qgis/QGIS-Enhancement-Proposals/issues/205"><strong>Port DB Manager Table Management Functionalities to Browser: SQL execution (part 3)</strong><span class="issue-shorthand"><strong> (#205)</strong></span></a></span> &#8211; <a href="https://lists.osgeo.org/pipermail/qgis-psc/2021-August/009437.html" rel="nofollow">Report</a><br />Besides SQL execution functionalities, an additional PR adds to QGIS core the query layer management tool that was provided by DB Manager plugin. The new API is fully covered by unit tests.</li>
<li class="task-list-item enabled position-relative border-right-0"><span class="reference"><a class="issue-link js-issue-link" href="https://github.com/qgis/QGIS-Enhancement-Proposals/issues/210"><strong>Locale support for numeric input and display: revision and enhancements<span class="issue-shorthand"> (#210)</span></strong></a></span> &#8211; <a href="https://lists.osgeo.org/pipermail/qgis-psc/2022-January/009622.html" rel="nofollow">Report</a><br />The work has been completed with multiple pull requests that fixed all localization issues that have been reported plus countless unreported issues that have been identified along the way.</li>
<li class="task-list-item enabled position-relative border-right-0"><span class="reference"><a class="issue-link js-issue-link" href="https://github.com/qgis/QGIS-Enhancement-Proposals/issues/217"><strong>Integrate GPS Tools plugin functionality into core QGIS<span class="issue-shorthand"> (#217)</span></strong></a></span> &#8211; <a href="https://lists.osgeo.org/pipermail/qgis-psc/2021-August/009436.html" rel="nofollow">Report</a><br />This grant sees the removal of the old, unmaintained &#8220;GPS Tools&#8221; core plugin, with all functionality from the plugin moved to reusable Processing algorithms or the unified Data Source Manager dialog. Since the functionality now uses the Processing framework, users gain the ability to run these tools in batch modes, as part of graphical models, and from 3rd party scripts and plugins. As a bonus the new tools are all fully covered by unit tests.</li>
<li class="task-list-item enabled position-relative border-right-0"><span class="reference"><a class="issue-link js-issue-link" href="https://github.com/qgis/QGIS-Enhancement-Proposals/issues/212"><strong>QGIS Server, OGC tests and Continuous Integration: OGC API Features (part 2 <span class="issue-shorthand">(#212)</span></strong></a></span> &#8211; <a href="https://lists.osgeo.org/pipermail/qgis-psc/2022-January/009625.html" rel="nofollow">Report</a><br />Thanks to the QEP funding, the OGC API Features standard for QGIS Server is<br />now checked in QGIS continuous integration since end-November 2021.</li>
<li class="task-list-item enabled position-relative border-right-0"><strong><span class="reference"><a class="issue-link js-issue-link" href="https://github.com/qgis/QGIS-Enhancement-Proposals/issues/215">Fixing terrain and camera issues in 3D<span class="issue-shorthand"> (#215)</span></a></span></strong> &#8211; <a href="https://lists.osgeo.org/pipermail/qgis-psc/2022-January/009621.html" rel="nofollow">Report</a><br />These improvements should make the 3D map view easier to use. Especially the camera control issues (unintuitivie camera rotation and wrong center point) were quite tricky to fix.</li>
<li><span class="reference"><a class="issue-link js-issue-link" href="https://github.com/qgis/QGIS-Enhancement-Proposals/issues/219"><strong>Review process on plugins.qgis.org and improvements<span class="issue-shorthand"> (#219)</span></strong></a></span> &#8211; This proposal has been withdrawn.</li>
</ol>
<p>Thank you to everyone who participated and made this round of grants a great success and thank you to all our sponsor and donors who make this initiative possible!</p>
