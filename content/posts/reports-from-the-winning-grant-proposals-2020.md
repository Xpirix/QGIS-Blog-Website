<p>With the <a href="https://blog.qgis.org/2020/07/10/qgis-grant-programme-2020-results/">QGIS Grant Programme 2020</a>, we were able to support ten proposals that were aimed to improve the QGIS project, including software, infrastructure, and documentation. The following reports summarize the work performed in the proposals. We&#8217;ll update this blog post <a href="https://github.com/qgis/PSC/issues/43">as more reports come</a> in:</p>
<ol>
<li><strong><a href="https://lists.osgeo.org/pipermail/qgis-psc/2020-December/009186.html">Quality Assurance methodology and infrastructure</a> (Alexandre Neto, Alexander Bruy, Giovanni Manghi)<br /></strong>
<p>The <a href="https://plugins.qgis.org/plugins/qgistester/"><strong>Tester plugin</strong></a> has been updated to run on QGIS 3.x. It allows to run automated and semi-automated tests and helps to conduct testing by providing step-by-step instructions to perform manual or verification tasks. An initial small set of tests for QGIS core functionality has been implemented as a separate <a href="https://plugins.qgis.org/plugins/coretests/"><strong>QGIS Core Tests plugin</strong></a>. Furthermore, a <a href="https://qgis.tenant.kiwitcms.org/"><strong>test management system</strong></a> and test plan based on KIWI TCMS has been set up and <a href="https://github.com/qcooperative/qgis-testing"><strong>documentation for testers</strong></a> has been created and published.</p>
</li>
<li><strong><a href="https://lists.osgeo.org/pipermail/qgis-psc/2021-January/009211.html">Smarter map redraws + tile download manager</a> (Martin Dobias)</strong>
<p><a href="https://github.com/qgis/QGIS/pull/40960">Smarter Map Redraws</a> avoid the annoying flicker when map in the map canvas is zoomed or moved. It is especially noticeable with background maps. The work has reduced the problem especially for raster layers. See the <a href="https://twitter.com/lutraconsulting/status/1349638508877590530">videos of comparison before/after</a>.<br /><a href="https://github.com/qgis/QGIS/pull/40998">Tile Download Manager</a> is not going to be very visible to the users, but it should make QGIS behave nicely with remote servers &#8211; until now it would be common that QGIS would request raster/vector tiles, then abort the requests while they were in progress when map got moved/zoomed, only to start those requests again &#8211; this should be avoided now.</p>
</li>
<li><strong><a href="https://lists.osgeo.org/pipermail/qgis-psc/2020-September/009084.html">DB Manager Table Management Functionalities to Browser Port &#8211; part 2</a> (Alessandro Pasotti)</strong>
<p>QGIS browser now exposes a new &#8220;Fields&#8221; item for vector layers that can be expanded to show the underlying fields, an icon identifies the base field type. New context menu items allow user to <strong>create and delete fields</strong>. At the connection level, a new context menu item allows you to create a <strong>new table</strong> for all DB connections that support the Connections API (PG, Spatialite, GPKG, MSSQL). All the new functions are implemented using the <strong>new connections API</strong> and <strong>exposed to Python</strong> for plugins/scripts. There have been many other small improvements in the API and in the browser, such as homogenization of the error/warning/success reporting .</p>
</li>
<li><strong><a href="https://lists.osgeo.org/pipermail/qgis-psc/2020-November/009144.html">QGIS Server, OGC tests and Continuous Integration</a> (Paul Blottiere)</strong><br />A Python tool named <strong>pyogctest</strong> has been implemented to run OGC tests in command line for the WMS 1.3.0 testsuite and has been integrated with GitHub Action in QGIS continuous integration mechanism to avoid regressions. The <strong>documentation</strong> chapter about OGC and conformance tests is now up-to-date with an explanation of how pyogctest can be used locally for server developers. Moreover, pyogctest is now also integrated with <strong>QGIS-Server-CertifSuite</strong> for the nightly tests. This way we have an homogeneous testing environment with CI. </li>
<li><strong><a href="https://lists.osgeo.org/pipermail/qgis-psc/2020-November/009146.html">QGIS Server performance monitoring</a> (Paul Blottiere)</strong><br />The whole QGIS-Server-PerfSuite has been upgraded to use 3.10 and 3.14 releases side by side with 2.18 and master branch. Performances may be now monitored daily with the latest releases. Moreover, a simple anomalies detection mechanism has been implemented and a mail is sent if a regression is detected. Several scenarios have been added to compare performance with the same data but relying on different providers (PostGIS, Spatialite, Geopackage and Shapefile). Finally, a simple mechanism based on multiprocessing has been implemented to simulate multi-clients situation. </li>
<li><strong><a href="https://lists.osgeo.org/pipermail/qgis-psc/2020-September/009078.html">FileGeodatabase spatial index in OpenFileGDB driver</a> (</strong><b>Even Rouault)<br /></b>This work has been successfully <a href="https://github.com/OSGeo/gdal/pull/2771">completed in <strong>GDAL master</strong></a> (for GDAL 3.2) and automatically benefits QGIS when it uses the OpenFileGDB driver. Performance-wise, for example, counting the number of features intersecting a spatial filter which returns 81 046 polygons, now runs in 400 ms with GDAL 3.2dev and the OpenFileGDB driver, versus 6.7 s before (full scan), vs 890 ms with the FileGDB driver (with FileGDB SDK 1.5). Interactive display in QGIS with the OpenFileGDB driver is as fluid as with the FileGDB one. Comparing behaviour of OpenFileGDB and FileGDB drivers with strace shows that they read a similar amount of data in the .spx file, which confirms it is uses correctly. The <strong><a href="https://github.com/rouault/dump_gdbtable/wiki/FGDB-Spec#specification-of-spx-files">filegdb reverse engineered specification</a></strong> was also updated.<b><br /></b></li>
<li><strong><a href="https://lists.osgeo.org/pipermail/qgis-psc/2020-September/009074.html">MacOS packages</a> (Peter Petrik)<br /></strong>All tasks from the proposal except the notarization process have been addressed since the work necessary to address critical bugs in projection, grass, saga, gdal, python and other parts of the MacOS packages exceeded expectations. (A note about the workaround for notarization has been added to the QGIS.org webpage for now.) Key improvements for QGIS 3.16 MacOS Packages are: QGIS-Mac-Packager without homebrew dependencies, updated GDAL3, PROJ6 &amp; GRASS 7.8.2, fixed Grass, Saga &amp;, GDAL provider loading, and many more. <strong><br /></strong></li>
<li><a href="https://lists.osgeo.org/pipermail/qgis-psc/2021-October/009535.html"><strong>Evaluate Qt for Python</strong></a><strong> (Denis Rouzaud)</strong><br />The initial evaluation was followed by a report on the migration to Qt-for-Python. The report&#8217;s recommendations are now being discussed in <a href="https://github.com/qgis/QGIS-Enhancement-Proposals/issues/237">QEP#237</a>.</li>
<li><a href="https://lists.osgeo.org/pipermail/qgis-psc/2022-January/009670.html"><strong>Settings registry</strong></a> <strong>(Denis Rouzaud)<br /></strong>The complete implementation of the core part has been achieved (settings, registry and Python bindings). All core settings were migrated. Other settings still have to be migrated, CI tests should be added to avoid usage of the old API and potential GUI improvements are outlined in the report.<strong><br /></strong></li>
<li><strong><a href="https://lists.osgeo.org/pipermail/qgis-psc/2022-May/009772.html">Vectorization of selective masking</a> (Julien Cabieces) </strong><br />Selective masking now avoids rasterization as much as possible. </li>
</ol>
<p>Thank you to everyone who participated and made this round of grants a great success and thank you to all our sponsor and donors who make this initiative possible!</p>
