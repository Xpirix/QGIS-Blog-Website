<p>A note from Jürgen Fischer, our release manager, heralds the start of preparations for QGIS 2.12. We depend all users out there to test it and report issues so we can make the best possible release. An edit version of the email to the developer mailing list from Jürgen follows:</p>
<p><em>Hi,</em></p>
<p><em>I&#8217;d like to remind everyone that the 2.11 development cycle ended yesterday at</em><br />
<em> 12:00 UTC. We are now in feature freeze on the road to the 2.12 release on 2015-10-23</em></p>
<p>Now <strong>we (</strong>the community) need to prepare QGIS for the release. No new features will be added anymore.</p>
<ul>
<li><strong>Users</strong>, if not already begun, should now start<br />
extensive testing of master and report bugs on the <a href="http://hub.qgis.org/">QGIS hub</a>.</li>
<li><strong>Developers</strong> should move their focus from creating new features to fixing bugs.</li>
<li><strong>Translators</strong> can can continue their work (we maintain all our translations on transifex now).</li>
</ul>
<p>The nightly builds of QGIS testing available for Windows (<a href="http://www.qgis.org/en/site/forusers/alldownloads.html#windows">download here</a>), Linux (<a href="http://www.qgis.org/en/site/forusers/alldownloads.html#debian-ubuntu">Debian and Ubuntu</a>) and Mac<br />
OS X (<a href="http://www.qgis.org/en/site/forusers/alldownloads.html#qgis-macos-testing">download here</a>) are now effectively snapshots of what&#8217;s going to be released.  Except of course<br />
for the bugs that are going to be fixed until release day.  For Windows there will also be <a href="http://www.qgis.org/downloads/weekly/">weekly release candidates</a> of the standalone installer. Lets keep up working together to make 2.12 another great release.</p>
<p>Jürgen</p>
